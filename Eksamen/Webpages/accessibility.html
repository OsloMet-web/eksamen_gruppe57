Site-Wide Accessibility

  • We have made sure to use semantic HTML elements on all webpages to ensure good accessibility and SEO (Search Engine Optimization)
  • Added the alt attribute to all images, describing the contents of the image if the image doesn't appear
  • Made sure all text is readable by adjusting font sizes, font type and color of the font
  • There are clear contrasts between text and background color
  • All colors on the website are picked so that they are pleasing to look at while making sure that all content is clearly visible for everyone, including the colorblind
  • We have made sure that the webpages work on different screen resolutions
  • Put all our webpages through a WAVE accessibility test to ensure that our site is up to standard
  • Added tabindex where deemed necessary to enhance the experience for users that only use a keyboard

Navigation bar

This navigation bar is created mainly for PC-screens and other wide forms of screens. We have ensured that all the elements are clearly visible and that these elements will change color to white when hovered over, making it easy for the user to see where their mouse is and what they are about to click.

The accessibility test shows an alert on the Campus navigation element, this is because the "OsloMet" logo beside it is also clickable and will take the user to the Campus page. The accessibility test shows that this is a redundant link, but we think of it rather as a feature where if the user is confused, clicking the logo will always take them back to the main page.

The dropdown menu is only available when the screen width decreases past 1080 pixels, it will show up as a clickable "hamburger-menu". The purpose of the dropdown menu is that it fits thinner screens. We have avoided the use of JavaScript by using a checkbox and a label to create a clickable area that serves as our dropdown menu.

Campus

This is the introduction to the website. The page is made deliberately so that the user knows exactly what to look at and it pushes the focus towards the content. The content of the page is therefore centered and we have also consciously left whitespace on both sides of the content to balance our visual design and enhance the visual experience for the user.

Topical

On this page we've made sure the text is easily readable at all screen widths, the clickable links containing more information appear at the bottom of the screen when the screen-width decreases to increase readability. Furthermore when the links are hovered over they change color to indicate to the user what their mouse is hovering over. This page is created with a focus on the information on the screen, avoiding other "flashy" content to engage the users focus onto the information.

Library-page

For the library page we use lorem ipsum text as the main content. but the header and subheaders are relevant. For the text on top of the image we added a black see-through background to ensure a good contrast between text and background, this improves readability substantially. Furthermore when the screens width decreases we have decided to put the image, picturing the students, at the bottom of the page to make sure that all the content is clearly visible regardless of what type of screen the user is using.

Eateries

When developing the eateries page, we focused on presenting the eateries and cafes in the most organized and uncomplicated manner. Our primary focus was on being informativ through illustrative Images of each place, title, location with link, clear description of the place and opening hours. When it comes to the icones underneath, they are mean to illustrate that the each place offers vegetarain and gluten free options, as well as one with a coffe service. The tabindex was added to each Eatery for easier navigation.
